Added a page on [consistent help](/spec/accessibility/consistent-help/), WCAG 2.2 Success Criterion 3.2.6. When a contact link, phone number, chat widget or FAQ is repeated across a set of pages, it has to occur in the same order relative to the rest of the content — the failures cluster on checkout, login and error pages, which drop the shared layout precisely where a stuck visitor reaches for support. The criterion never obliges you to offer help in the first place; it only constrains where the help you already offer sits.
